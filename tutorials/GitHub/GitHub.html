Detailed Step-by-Step Guide

Complete walkthrough matching the video

Part 1: Organise Your Local Files

Before uploading to GitHub, organise the files from your previous tutorials. Instead of loose .html files, put each project inside its own folder and rename the file to index.html.

How to Organise

  1. Open your AI Projects folder (the one from earlier tutorials)
  2. Create a new folder for each project (e.g. retro-game-gemini)
  3. Drag the corresponding .html file into the folder
  4. Rename the file to index.html

Here's what your AI Projects folder should look like after organising:

ai-projects/
  helloworld/
    index.html
  retro-game-claude/
    index.html
  retro-game-gemini/
    index.html
  space-invaders-gemini/
    index.html

Each project gets its own folder, and every file is renamed to index.html. Each of these folders can become its own GitHub repository and live website.

Why Rename to index.html?

Naming your file index.html means your URL stays clean. Instead of username.github.io/project/retrogame.html, you get username.github.io/project/. This is a web convention: browsers automatically look for index.html in any folder.

Part 2: Create the Repository on GitHub

Step 1: Go to github.com/new

This takes you straight to the "Create a new repository" page. Name your repository to match your local folder name (e.g. retro-game-gemini). Leave everything else as default (Public is fine). Click Create repository.

Step 2: Create the file in the blue box

After creating the repository, you'll see a page with a blue "Quick setup" box at the top. This is the key area:

Look for the Link Inside the Blue Box

Below the "Quick setup" heading, look for: "Get started by creating a new file or uploading an existing file." Click the "creating a new file" link. This is easy to miss!

Step 3: Paste your code

  1. In the filename field at the top, type index.html
  2. Open your local index.html in Sublime Text (right-click → Open with Sublime Text)
  3. Copy the code (Cmd+C / Ctrl+C). Make sure to copy, not cut, so you keep the local copy.
  4. Paste into the GitHub editor (Cmd+V / Ctrl+V)

Step 4: Commit changes

  1. Click the green Commit changes... button
  2. Copilot may suggest a commit message automatically. Otherwise, type initial commit
  3. Leave the extended description blank
  4. Click Commit changes again to confirm

Your code now lives on the cloud in your GitHub repository.

Part 3: Enable GitHub Pages

Step 1: Open Settings

From your repository page, click the Settings tab at the top.

Step 2: Navigate to Pages

In the left-hand sidebar, scroll down and click Pages.

Step 3: Set the branch

  1. Under Branch, click the dropdown that says None
  2. Select main
  3. Click Save

Wait about 60 to 90 seconds, then refresh the page. Your live URL will appear at the top.

Part 4: Visit Your Live Site

After refreshing, you'll see a banner with your live URL:

Click Visit site. Your project is now live on the internet, playable and shareable with anyone!

Well Done!

You went from generating code in your LLM, to saving it locally, to deploying it on the internet. This workflow will become second nature. Repeat it for all your projects!

Troubleshooting

Common problems and solutions

Pro Tip: Use AI for Help

Stuck? Describe your problem to your LLM of choice (ChatGPT, Claude, Gemini). You can even share screenshots; AI can read images!

Common Issues

"404 - Page Not Found"

  • Wait about 60 to 90 seconds; GitHub Pages needs time to build your site
  • Make sure your file is named exactly index.html (not Index.html or index.HTML)
  • Check that GitHub Pages is enabled in Settings → Pages
  • Make sure the Branch is set to main (not "None")

"I can't find the 'creating a new file' link"

Look inside the blue "Quick setup" box that appears after creating an empty repository. The link is below the heading text. If your repository already has files, click the Add file button instead, then select Create new file.

"I can't find the Settings or Pages option"

Make sure you're on your repository page (not your profile page). The URL should look like: github.com/username/myproject

"My changes aren't showing up"

  • Wait about 60 to 90 seconds after committing
  • Hard refresh your browser: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows)
  • Try opening in an incognito/private window

"The page shows raw code instead of my website"

You're looking at the GitHub repository view, not the GitHub Pages view. Your live site is at username.github.io/project/, not github.com/username/project.
