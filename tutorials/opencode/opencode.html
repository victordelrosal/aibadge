Quick Start: Install OpenCode

4 steps to your first free AI coding session

What is OpenCode?

OpenCode is a free, open-source AI coding assistant that runs inside your terminal. It can write code, explain code, fix errors, and help you learn, all from the command line, with several free models available through opencode.ai.

1
Open Your Terminal

Mac: open Terminal (Cmd+Space → type Terminal). Windows: open PowerShell (Start menu → type PowerShell).

2
Install OpenCode

Mac/Linux: paste the curl install line. Windows: install Node.js, then run npm i -g opencode-ai.

3
Launch It

Type opencode in the terminal and press Enter.

4
Connect a Free Model

Run /connect inside OpenCode, sign in via your browser, paste the key back, then /models → pick a free one.

One Terminal, Two Systems

Pick the section below that matches your machine: Mac uses a one-line curl install, Windows needs Node.js first then npm. The rest of the tutorial is identical for both.

Zero Cost

No credit card. No subscription. OpenCode is open source, and several capable models are free to use through OpenCode Zen.

Mac: Install OpenCode

One line, no Node.js required

Step 1: Open Terminal

Press Cmd+Space to open Spotlight, type Terminal, and press Enter.

Step 2: Run the Install Command

Copy and paste this line, then press Enter:

curl -fsSL https://opencode.ai/install | bash

The script downloads OpenCode and puts it on your PATH automatically. It usually finishes in under a minute.

Prefer Homebrew?

If you already use Homebrew, you can install via brew install anomalyco/tap/opencode instead. Either way works.

Step 3: Launch OpenCode

opencode

If you see the OpenCode prompt, you're in. Jump down to Connect to a Free AI Model below.

"command not found: opencode"

Close Terminal completely (Cmd+Q), reopen it, and try opencode again. Your shell needs a fresh window to pick up the new PATH.

Windows: Install OpenCode

Node.js + one npm command

Step 1: Open PowerShell

Press the Windows key, type PowerShell, and click it. A blue window opens.

Step 2: Check if Node.js is Already Installed

Type this and press Enter:

node --version

What did you see?

A version number (like v20.11.0)? Skip to Step 4. "not recognized"? Do Step 3 first.

Step 3: Install Node.js (only if Step 2 failed)

  1. Go to nodejs.org
  2. Click the big green LTS button
  3. Run the installer (Next → Next → Finish)
  4. Close PowerShell completely
  5. Reopen PowerShell (this step matters: the window needs to be new)
  6. Run node --version again to confirm a version prints

Step 4: Install OpenCode

npm i -g opencode-ai

The -g flag installs OpenCode globally, so you can run it from any folder.

Step 5: Launch OpenCode

opencode

You should see the OpenCode interface. Continue to Connect to a Free AI Model below.

Connect to a Free AI Model

Sign in once, then start coding

Once OpenCode is running, you'll see a prompt at the bottom of the terminal. The commands below are typed inside OpenCode itself (not the regular shell).

Step 1: Start the Connection Flow

/connect

Step 2: Pick a Provider

From the list, choose OpenCode Zen. This is the curated set of free models that just work.

Step 3: Sign In via Browser

OpenCode opens a browser tab. Sign in with Google or GitHub. After you authenticate, your API key is shown on the page.

Step 4: Paste the Key Back

Copy the API key from the browser and paste it back into the terminal where OpenCode is waiting. Press Enter.

Step 5: Choose a Model

/models

Pick a free model from the list. Grok Code Fast 1 is a great default. Press Enter and you're ready.

You're In

Type any coding question or instruction at the prompt and OpenCode will answer. Try: "Write a Python script that prints hello world."

Optional: Let OpenCode Read Your Project

If you've opened OpenCode inside an existing code folder, run this once to let it scan and remember the project structure:

/init

It creates an AGENTS.md file so OpenCode has context for future sessions.

Free Models Worth Trying

Pick one in /models

OpenCode Zen rotates the free model line-up over time. These have been reliable picks:

Switch Anytime

Type /models again at any time to swap. Different models are stronger at different things, so it's worth trying a few.

Troubleshooting

Common problems and fixes

Pro Tip: Use AI for Help

Stuck? Paste the exact error into ChatGPT, Claude, or Gemini and describe what you were doing. You can even screenshot the terminal: AI can read images.

Common Issues

'npm' is not recognized (Windows)

Node.js isn't installed yet, or PowerShell hasn't picked it up. Go back to Windows Step 3, install Node.js, then close and reopen PowerShell.

'node' is not recognized (Windows)

Same fix: close PowerShell completely and reopen it. New windows pick up the updated PATH.

'opencode' is not recognized / command not found

Close the terminal completely, reopen, and try opencode again. If that still fails, re-run the install command from the section that matched your OS.

Execution policy error (Windows)

PowerShell sometimes blocks scripts. Run this once, then retry:

Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

EACCES: permission denied (Mac)

npm can't write to a system folder. Run the install with sudo:

sudo npm i -g opencode-ai

It'll ask for your Mac login password. Type it and press Enter; the password won't show as you type, that's normal.

/connect isn't doing anything

Check your internet connection, then try again. If a browser tab didn't open, copy the URL OpenCode prints into a browser manually.

Browser opens but no API key appears

Make sure you completed the sign-in (Google or GitHub) all the way through. Reload the page if the screen looks stuck, then re-run /connect.

"Model not available" or rate-limit message

Free models are popular and occasionally throttle. Run /models and pick a different one from the list, then try again.
