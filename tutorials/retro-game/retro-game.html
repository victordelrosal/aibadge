Detailed Step-by-Step Guide

Complete instructions for both exercises

Part 1: Code a Simple Retro Game

Step 1: Open Your LLM

For this exercise, try using a different LLM from the one you used in Hello World. The tutorial demonstrates both claude.ai and gemini.google.com side by side. If you're happy with ChatGPT, you can stay there, but it's a good idea to explore others.

Step 2: Enter the Prompt

Copy and paste this prompt. The tech stack is the same as before: HTML, JavaScript, and CSS in a single file.

Code a simple and fun retro game. Output as single-file HTML, JS, CSS.

Step 3: Copy and Save the Code

Copy the generated code. Open Sublime Text, paste it, and check for the opening and closing HTML tags. Save as retrogame.html in your AI Projects folder.

Important

Remember: the file must end with .html otherwise your browser won't recognise it as a webpage.

Step 4: Run the Game

Open your browser and use Cmd+O (Mac) or Ctrl+O (Windows) to open the file. Navigate to your AI Projects folder and select the file. Your game should load and be playable immediately.

Nice!

You now have a playable retro game generated entirely by AI. Take a moment to play it!

Step 5: Talk to the Code

In the same conversation, ask the LLM to make improvements. For example:

  • "Change the background colours"
  • "Change the colour palette"
  • "Make the game faster"
  • "Add a score counter"

Each time, copy the new code, replace everything in Sublime Text (Cmd+A, delete, paste), save, and refresh the browser.

Part 2: Code Space Invaders

Step 1: Enter the Prompt

You can ask for a specific game. Copy this prompt into your LLM:

Code Space Invaders. Output as single-file HTML, JS, CSS.

Step 2: Save and Run

Copy the code, open a new tab in Sublime Text (Cmd+N / Ctrl+N), paste the code, and check the HTML tags at the top and bottom. Save as spaceInvaders.html in your AI Projects folder. Open it in your browser to play.

Step 3: Personalise and Iterate

Keep the conversation going. Ask for changes and keep saving and refreshing:

  • "Add power-ups that drop from destroyed enemies"
  • "Make the aliens move faster each wave"
  • "Add a high score display"
  • "Change the ship to a different design"

Troubleshooting

Common problems and solutions

Pro Tip: Use AI as Your IT Support

Game not working? Take a screenshot of the problem and paste it directly into your LLM conversation. Tell it "the game doesn't work" and it will try to fix the code for you. This is a great skill for debugging.

Common Issues

"Game doesn't load or shows a blank screen"

The generated code may have a bug. Take a screenshot of what you see and paste it back into the LLM conversation with "the game doesn't work." It will analyse the issue and generate fixed code.

"Game crashes when I click to start"

Some LLMs generate code where game objects aren't initialised until you click, but the draw function runs immediately. Send a screenshot back to the LLM and it will fix the initialisation order.

"File opens as text, not a game"

File wasn't saved with .html extension. Rename the file or save again with the correct extension.

"Can't find my file"

Search for the filename in Finder (Mac) or File Explorer (Windows). Make sure you saved it in your AI Projects folder.

Common Mistakes to Avoid

  • Saving as retrogame.txt instead of retrogame.html
  • Not copying ALL the code (must start with <!DOCTYPE html> or <html>)
  • When replacing code, forgetting to Select All (Cmd+A) before pasting the new version
