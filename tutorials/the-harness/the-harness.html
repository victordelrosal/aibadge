Quick Start: run the harness experiment

Feel the difference context makes, in four steps

This lesson has one big idea: the model is not what changes your results, the context is. You will run the same AI on the same business twice, and watch the output go from generic to genuinely good, just by feeding it a richer brief (the harness).

The prompt that starts each run

You are the Octopus orchestrator. Read context.md in full, then run the pipeline end to end to deliver everything it asks for. Do not ask me questions; decide and build.

What you are proving

You are not learning a new model. You are learning to build the harness that makes any model do its best work: the single highest-leverage skill in this whole course.

The two exercises

Inkwell (class) and FlexHaus (individual), with the full pack to download

What is in the harness

A rich context.md (the harness) carries everything the AI needs to do world-class work: a backstory, a real dataset to reason from, named specialist agents (not generic ones), clear brand direction, and an exact artifact spec (here, a three.js page) plus a cold-verify gate. The empty brief has none of that. Run both; the difference is undeniable.

Exercise 1: Inkwell Books (class exercise)

The brief, exactly as shown in class

Inkwell is a 9-branch independent bookshop chain. Footfall is down 28% in two years, its loyalty scheme is still a paper stamp card, and it has no email list. Budget: 5,000 euros and six weeks before Christmas. Mission: produce a launch-ready re-engagement plan with one working artifact. They would like you to: - Diagnose why younger customers left and size the opportunity - Design a re-engagement concept - Build a real sign-up landing page - Draft the launch email and three social posts Use Octopus to orchestrate the pipeline and manage the end-to-end process.

The dataset: Inkwell has no CRM, but every till logs sales, so the rich harness ships an anonymised point-of-sale export (~30,600 rows). Your analyst agent must aggregate it to find why under-35s left.

Exercise 2: FlexHaus Gym (individual exercise)

The brief, exactly as shown in class

FlexHaus is a single-location boutique gym with 600 members. Monthly churn is 9%, class attendance is patchy, and 40% of members never book a second class. Budget: 2,000 euros and one month. Mission: produce a member-retention plan with one working artifact. They would like you to: - Diagnose the churn and find the at-risk segment - Design a retention concept - Build a real booking or sign-up page that includes top-notch sales copy to win members back Use Octopus to orchestrate the pipeline end-to-end. Submit it via Moodle as a GitHub page URL.

The dataset: a gym has a member system, so the rich harness ships the full class-bookings export (~45,300 rows). There is no member table: your analyst must GROUP BY member_id to rebuild members and model churn.

How to run each exercise

  1. Make a folder and drop in the empty context.md. Point Octopus at it with the prompt above. Save the output.
  2. Start fresh. Use the rich context.md and put the CSV beside it. Run again.
  3. Compare the diagnosis, the page, and the copy. Notice the rich run cites real numbers; the empty run invents them.
  4. Catch the octopus lying: find one moment it got something wrong (a made-up stat, a dropped handoff, an off-brief page). That catch is the point.

Use only the data in the file

Every figure in these datasets is synthetic, made for teaching. Do not treat Inkwell or FlexHaus as real businesses, and never claim a number the file does not support.

Troubleshooting

Common snags and fixes

Pro Tip: use AI as your IT support

Stuck on an error? Paste the message straight into ChatGPT, Claude, or your terminal agent and ask it to walk you through the fix.

Common issues

"The empty run actually looks fine"

Look closer: are its numbers real or invented? Is the page generic enough to belong to any business? That is exactly the failure the harness fixes.

"The agent will not read the CSV"

Keep the CSV in the same folder as context.md and name it exactly as the brief references. Tell the agent to load and aggregate it before writing the diagnosis.

"My three.js page is blank"

Open the browser console and read the first error. A missing import or a WebGL failure is the usual cause; the brief asks for a no-WebGL fallback, so build that too.

"Octopus is not installed"

Follow the Octopus install steps at victordelrosal.com/octopus, or pair this with the Five Innovators lesson first.
