Detailed Guide

Why folders, why markdown, and what to put in each file

Why folders matter (the folder is, in part, the agent)

Modern AI tools that read your machine (Claude Code, ChatGPT Desktop, Cursor, Gemini for Workspace) infer what you are working on from your folder names and structure. When you drag a file into a chat, the assistant uses where it lives in your tree to make sense of it. Tidy tree, tidy answers. And future you finds things in seconds. The cheapest productivity upgrade in your life is a folder you can name out loud.

Why markdown (AI's native tongue)

By 2026, markdown is the lingua franca for agents. Three reasons:

  • Parsed better, fewer tokens. LLMs read clean markdown more accurately than .docx, .pdf, or rich text, and burn fewer tokens doing it.
  • Native output of every modern AI. ChatGPT, Claude, Gemini, Copilot all produce markdown by default. Meeting it in its own format compounds.
  • Plain text. Forever. It is just # heading, - bullet, **bold**. Opens anywhere, breaks nowhere, survives every tool change.

Markdown vs plain text files

.txt

Best for: raw notes, logs, transcripts, simple dumps. Universal, minimal.

Cons: harder to scan, weak hierarchy, code and instructions blur together.

.md

Best for: agent instructions, specs, READMEs, plans, architecture notes. Human + machine readable.

Cons: a tiny bit of formatting discipline; markdown flavours can differ slightly.

The three files that earn their keep

Every project starts with these. They are short on purpose. Five lines beats five hundred.

README.md — what this folder is

# My AI Workspace

A small home for the AI tools, personas, and notes I am building.

Owner: [your name]
Started: [today's date]

## What lives here
- personas/ — my AI teammates (filled in Lesson 8)
- skills/   — reusable recipes my teammates can run (Lesson 9)
- notes/    — scratchpad for ideas, links, half-thoughts

CLAUDE.md — always-on context for any AI

Named CLAUDE.md by convention (Claude Code reads it automatically). If you use Gemini, Cursor, or ChatGPT desktop, name it whatever that tool reads (e.g. INSTRUCTIONS.md) or just keep CLAUDE.md and paste its contents into a custom GPT or Gem. Same idea either way: a short note any AI can read about how to work with you.

# Working with me

- Be direct, be warm.
- I run [a small literacy non-profit / a one-person consultancy / a classroom of 30 students].
- Default tone: encouraging, plain language. No jargon unless I ask.
- No em dashes. Use commas, semicolons, or full stops.
- When you propose options, give me your top recommendation first.
- If you are uncertain, say so. Don't fabricate.

Troubleshooting

Common snags and fixes

Pro tip: AI as your IT support

Stuck on any of the steps below? Paste the exact error or the screen you are looking at into ChatGPT or Claude with: "I am on macOS [version] / Windows 11. I am trying to create a markdown file. Here is what I see: [paste]. Walk me through the next click." The assistant will guide you through.

My file saved as README.md.txt

Windows Notepad and Mac TextEdit sometimes append .txt automatically. On Mac TextEdit, do Format → Make Plain Text first, then save. On Windows Notepad, in the Save dialog change Save as type from Text Documents (*.txt) to All Files, then type the full name README.md.

I cannot see file extensions in Finder/Explorer

On Mac: Finder → Settings → Advanced → tick Show all filename extensions. On Windows 11: open File Explorer, click the View menu → Show → tick File name extensions. Without this, you cannot tell README.md from README.md.txt.

What editor should I use to edit markdown?

Any plain text editor works. Free options worth a look: Obsidian, Typora, VS Code. But honestly: TextEdit (Mac) or Notepad (Windows) is fine. Markdown is just text.

Should the file be CLAUDE.md or something else?

If you use Claude or Claude Code, keep CLAUDE.md: it is the convention Claude reads automatically. If you use Gemini, Cursor, or a ChatGPT custom GPT, the name does not matter for the AI; it matters for you. Pick one name and stick with it. The contents are what counts.

Where should I put the workspace folder?

Anywhere you can find it again. Documents folder is fine. Desktop is fine. iCloud Drive or Dropbox is fine if you want it on multiple machines. Avoid putting it inside a system folder (Applications, Library) where it will get lost.

Common Mistakes to Avoid

  • Saving as .docx, .pages, or .notes — AI parses these poorly. Markdown only.
  • Writing 500-line README files — the point is short and scannable. Five lines beats fifty.
  • Filling personas/ and skills/ with random files now — they fill themselves in Lessons 8 and 9.
  • Treating this as one-and-done — the workspace grows. Edit your CLAUDE.md as you learn what works.

Why this lesson, and where it fits

One of the more exciting use cases of AI in 2026 is AI personas that remember who you are: the context, the type of work you do, the specific functions, tasks, and roles you perform day to day, whether for your job, a hobby, or another interest. Previously we looked at creating archetypes, the five innovator archetypes. We went through all four in the last exercise and reserved the Manager role for you, the human driving these agents. Now we want to move on to the more agentic use of AI, where you create personas customised for you. But before we do that, we need to set up the structure and understand how it works.

Why folders: the AI feels forgetful, but it is not the AI

Sometimes we feel like we are repeating ourselves to AI. It feels forgetful. Some providers have solved part of this with persistent memory, but I find that is more useful at a project level. The way to solve it is a folder structure: scattered ideas become organised ideas, one home per project or per goal. The folder is, in part, the agent, because it gives it context. We are going to see how AI tools read what you have stored on your local machine, and how you can paste and carry that context across different sessions. It also makes finding things in the future a matter of seconds.

Markdown: the native tongue of AI

The native file type AI uses is called markdown. It looks like a text document. The difference is the .md extension and a few small formatting conventions we will see. Plain text files are great for raw notes, logs, transcripts: universal, but harder for AI to scan, less hierarchy. Markdown files are useful for agent instructions, README files in repositories, and similar. They are human-readable too: headings, lists, code, all in plain text. That is the formatting AI prefers to read.

The structure we are going to build

We will replicate this structure on your own local computer. Earlier in the AI Badge we worked on a folder of mini-projects: Hello World, the games, AI Interviews You, and so on. What we want now is an outer my-ai-workspace folder where you create that structure first, and then drop in your project folders alongside it.

Inside that workspace, two key things to start with: a personas/ folder and a skills/ folder. We populate those in the next two tutorials. Then a README.md document and a CLAUDE.md document at the root. The CLAUDE name will vary if you are using Gemini or ChatGPT instead, but the idea is the same.

What goes in the README and CLAUDE files

The README.md is your workspace's front door: it gives context about who you are, what you do, who owns it, and what lives in each subfolder. It will vary for each person. CLAUDE.md is similar: it gives the wider context of who you are, what you do, your preferences. Think of the two as going hand in hand. The notes/ folder is for half-formed thoughts: ideas, observations, links, anything relevant to your working life or your projects.

The principle: context plus memory

The main principle to take away is that you are giving AI two things: context for the work you want to do, and memory. It keeps track of what has been said, what you have written. This is a powerful combination, because any time you start a new conversation, whether at the project-folder level, in a sub-project subfolder, or at the general level, the AI does not start from scratch. You do not have to re-explain who you are or what you do. It already knows. This is part of the scaffolding we are creating so AI can do its most powerful work. Think of it as a home for AI, for your AI team. We will see how to furnish it.
